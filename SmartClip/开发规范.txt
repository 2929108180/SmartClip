# 📋 产品需求文档：SmartClip

## 1. 产品愿景
开发一款 Windows 平台下**极致轻量、无感集成**的剪贴板管理工具。它将完全接管系统原生 `Win + V` 快捷键，提供比原生更快、更强（模糊搜索、置顶、持久化）的体验，同时保持极低的资源占用。
UI保持精美和现代化 但不要过于臃肿

## 2. 技术栈架构 (已确认)
*   **核心语言**：C# (.NET 9)
*   **编译模式**：ReadyToRun (追求极致启动速度和防反编译)
*   **数据存储**：JSON (元数据) + 文件系统 (图片缓存) ——
*   **系统交互**：Win32 API (User32.dll) 实现键盘钩子、剪贴板监听、模拟粘贴。
*   **UI选择**：WPF-UI 4.1.0架构，当前项目已引入

---

## 3. 功能需求 (Functional Requirements)

### 3.1 核心监听与捕获
*   **全格式监听**：不仅监听**纯文本**，还要监听**富文本(RTF/HTML)**（保留格式，如代码高亮）、**图片**（Bitmap）、**文件列表**（复制文件时只存路径，不存文件实体）。
*   **重复过滤**：连续复制相同内容时，自动合并或更新时间戳，不产生重复记录。
*   **来源记录**：(补充细节) 记录复制来源的应用程序（如：来自 Chrome、来自 VS Code），并在列表中显示小图标，方便快速识别（可先忽略作为优化项)。

### 3.2 交互与显示 (接管 Win+V)
*   **全局唤起**：使用底层键盘钩子拦截 `Win + V`，阻止系统原生菜单弹出，显示 SmartClip。
*   **跟随鼠标/光标**：窗口出现位置应智能判断：
    *   优先：文本输入光标 (Caret) 附近（模拟原生体验）。
    *   其次：鼠标指针附近。
    *   **补充细节**：必须处理**屏幕边缘检测**，防止窗口出现在屏幕外或任务栏遮挡处。多显示器环境下需在鼠标所在的显示器弹出。
*   **失去焦点行为**：用户点击窗口外部、切换程序或按下 `Esc` 时，窗口立即隐藏（而非关闭），释放前台内存。
*   **拖动窗口**：选中窗口顶部可以拖动窗口位置。

### 3.3 列表与排序逻辑
*   **分区排序算法**：
    1.  **Level 1 (置顶区)**：用户手动 Pin 的条目，永远在最上，按 Pin 时间倒序。
    2.  **Level 2 (高频区)**：(补充细节) 算法自动计算。过去 N 天内使用/粘贴次数超过 X 次的条目，显示在置顶区下方。
    3.  **Level 3 (历史区)**：按复制时间倒序排列（最新的在上面）。
*   **图片预览**：如前所述，使用缩略图流加载，不仅要支持 PNG/JPG，最好支持 **GIF 动图首帧预览**。

### 3.4 搜索功能
*   **模糊搜索**：支持拼音首字母（可选）、部分匹配。例如输入 "sql" 能搜到 "select * from..."。
*   **类型过滤**：(补充细节) 输入 `/img` 只显示图片，输入 `/file` 只显示文件记录。

### 3.5 粘贴操作
*   **智能粘贴**：用户双击条目或按回车时：
    1.  将该内容写入系统剪贴板。
    2.  隐藏 SmartClip 窗口。
    3.  **补充细节**：自动模拟 `Ctrl + V` 按键，直接将内容粘贴到用户刚才激活的窗口中（实现“一键上屏”，无需用户再按一次粘贴）。
    4、需要支持粘贴为纯文本，只对"纯文本"做空格处理，代码/富文本保持原样

## 4. 数据管理与存储 (Data & Storage)

### 4.1 存储策略
*   **History.json**：存储 ID、文本摘要、图片路径、置顶状态、引用计数、哈希值（用于去重）。
*   **Cache 目录**：仅存储图片文件（以哈希值命名，避免重复存储相同图片）。

### 4.2 清理机制 (GC) —— **至关重要**
由于没有数据库自动管理，必须手写清理逻辑，防止磁盘爆炸：
*   **启动/退出时检查**：
    *   **条数限制**：固定最多 300 条，超出时删除最旧的非置顶记录。
    *   **时间限制**：固定最多 30 天，删除超过限制天数的非置顶记录。
    *   **孤儿文件清理**：(补充细节) 扫描 Cache 目录，删除那些在 JSON 中没有被引用的图片文件。

### 💡 补充给开发者的提示 (坑点预警)

1.  **UI 线程阻塞**：保存图片到磁盘是 IO 操作，搜索是 CPU 操作。**务必使用 `Task.Run` 放到后台线程**，否则复制大图片时，你的剪贴板工具会卡死，甚至导致用户当前的复制操作变慢。
2.  **剪贴板占用冲突**：剪贴板是系统独占资源。如果你读取剪贴板时，另一个程序（如 Word）正在写入，会报错。**代码中必须加 `try-catch` 重试机制**。
3.  **高分屏适配 (DPI)**：WPF 对 DPI 支持很好，但获取鼠标位置的 `Win32 API` 返回的是物理像素，设置 Window 位置时可能需要除以 DPI 缩放比例，否则窗口位置会偏。

### 核心解法：不要“去获取”，要“监听变化”

你现在的逻辑可能是：

> 1.  用户打开剪贴板 $\rightarrow$ `IntPtr target = GetForegroundWindow();` (记住了窗口 A)
> 2.  用户固定剪贴板。
> 3.  用户点击窗口 B (窗口 B 激活)。
> 4.  用户点剪贴板粘贴 $\rightarrow$ 你的代码还在用第 1 步的变量 `target` (依然是 A) $\rightarrow$ **粘贴错误**。

**正确的“完美方案”逻辑：**

> 1.  程序启动，安装 **系统事件监听器 (WinEventHook)**。
> 2.  用户打开剪贴板，固定住。
> 3.  用户点击窗口 B $\rightarrow$ Windows 广播消息：“喂！现在的焦点变成 B 了！” $\rightarrow$ **你的监听器收到消息，悄悄把变量 `target` 更新为 B**。
> 4.  用户点击窗口 C $\rightarrow$ Windows 广播 $\rightarrow$ **变量 `target` 更新为 C**。
> 5.  用户点剪贴板粘贴 $\rightarrow$ 你的代码直接读取当前最新的变量 `target` (也就是 C) $\rightarrow$ **完美粘贴**。

### 为什么你觉得“不能实时获取”？

你可能是在剪贴板窗口的 `Activated` 事件，或者在粘贴按钮的 `Click` 事件里去调 `GetForegroundWindow`。

  * **如果你用了 `WS_EX_NOACTIVATE`（无激活）：**
    当你点击剪贴板按钮的那一瞬间，理论上焦点**并没有**转移给剪贴板，焦点**依然**在底下的窗口（例如微信）。此时直接调 `GetForegroundWindow` **应该**是能拿到的。

      * *如果拿不到*：说明你的 `NoActivate` 实现有问题，或者该控件（如 TextBox）强制抢走了焦点。

  * **如果你没用 `WS_EX_NOACTIVATE`：**
    点击剪贴板的一瞬间，焦点变成了“剪贴板自己”。这时候你调 API，当然只能拿到你自己，或者上一个缓存的句柄。

### 终极补丁：`GetGUIThreadInfo` (比 GetForegroundWindow 更强)

如果 `GetForegroundWindow` 让你失望了，你需要祭出 Windows API 的大杀器：**`GetGUIThreadInfo`**。

它不仅能告诉你哪个窗口是活动的，还能告诉你**光标（Caret）具体在哪个文本框里**。

#### C\# 实现思路

```csharp
// 1. 定义结构体
[StructLayout(LayoutKind.Sequential)]
public struct GUITHREADINFO
{
    public int cbSize;
    public int flags;
    public IntPtr hwndActive;   // 真正活动的窗口
    public IntPtr hwndFocus;    // 真正拥有键盘焦点的控件 (比如微信的输入框)
    public IntPtr hwndCapture;
    public IntPtr hwndMenuOwner;
    public IntPtr hwndMoveSize;
    public IntPtr hwndCaret;    // 光标位置
    public System.Drawing.Rectangle rcCaret;
}

[DllImport("user32.dll")]
public static extern bool GetGUIThreadInfo(uint idThread, ref GUITHREADINFO lpgui);

// 2. 获取真正的焦点逻辑
public IntPtr GetRealFocusWindow()
{
    GUITHREADINFO guiInfo = new GUITHREADINFO();
    guiInfo.cbSize = Marshal.SizeOf(guiInfo);
    
    // 传 0 代表获取“前台线程”的信息（也就是用户当前正在操作的那个线程）
    if (GetGUIThreadInfo(0, ref guiInfo))
    {
        // 如果 hwndFocus 不为空，说明有个输入框正等着输入
        if (guiInfo.hwndFocus != IntPtr.Zero)
            return guiInfo.hwndFocus;
            
        // 否则返回活动窗口
        return guiInfo.hwndActive;
    }
    return IntPtr.Zero;
}
```

### 总结

这个问题**绝对有解**。不要被“固定模式”迷惑了。

**你的行动清单：**

1.  **必须实现 `SetWinEventHook` (EVENT\_SYSTEM\_FOREGROUND)**：这是核心。不要在粘贴时才去问“谁是窗口”，而要时刻盯着“谁变成了窗口”。
2.  **在 Hook 回调里过滤自己**：如果通知说“现在的焦点是剪贴板”，**无视它**，不要更新你的 `LastTargetWindow` 变量。这样你的变量里永远存的是\*\*“除了剪贴板之外，用户最后操作的那个软件”\*\*。
3.  **粘贴时直接用变量**：点击粘贴时，直接向 `LastTargetWindow` 发送数据，甚至不需要把焦点切过去（使用 `PostMessage` 或 `UI Automation`）。

这套机制是 TeamViewer、向日葵、以及各种屏幕取词翻译软件的通用底层逻辑，完全可行。

### 问题描述
当 Windows 11 的搜索框或开始菜单打开时，SmartClip 剪贴板窗口会被系统窗口遮挡，即使设置了 TopMost 也无法显示在最前面。

**原因分析：**
Windows 的窗口层级分为多个"频段 (Bands)"：
1. 普通应用频段 (ZBID_DESKTOP)
2. 置顶应用频段 (TopMost) - SmartClip 在这里
3. 系统频段 (ZBID_UIACCESS / ZBID_SYSTEM_TOOLS) - 开始菜单、搜索框在这里

系统频段 > 置顶应用频段，这是微软的安全机制，防止恶意软件伪造系统 UI。

### 彻底解决方案：获取 UIAccess 特权

要让窗口真正浮在系统菜单之上，必须获取 UIAccess 特权。

#### 步骤 1：修改 app.manifest
```xml
<requestedExecutionLevel level="asInvoker" uiAccess="true" />
```

#### 步骤 2：对 EXE 进行数字签名

##### 方式一：购买商业代码签名证书（推荐用于商业软件）
- **DigiCert**：约 $500/年
- **Sectigo (Comodo)**：约 $200-400/年
- **GlobalSign**：约 $300/年
- 需要企业认证或个人身份验证

##### 方式二：使用免费/低成本证书
- **SignPath.io**：为开源项目提供免费签名服务（需审核）
- **Let's Encrypt + Azure SignTool**：技术上可行但较复杂
- **自签名证书**：免费，但用户需手动信任（体验差）

##### 方式三：自签名证书（开发/测试用）
```powershell
# 1. 创建自签名证书
$cert = New-SelfSignedCertificate -Type CodeSigningCert -Subject "CN=SmartClip Dev" -CertStoreLocation Cert:\CurrentUser\My

# 2. 导出证书
$pwd = ConvertTo-SecureString -String "password" -Force -AsPlainText
Export-PfxCertificate -Cert $cert -FilePath "SmartClip.pfx" -Password $pwd

# 3. 签名 EXE
signtool sign /f SmartClip.pfx /p password /fd SHA256 SmartClip.exe

# 4. 安装证书到受信任的根证书颁发机构（需管理员权限）
Import-Certificate -FilePath "SmartClip.cer" -CertStoreLocation Cert:\LocalMachine\Root
```

#### 步骤 3：安装到受信任目录
UIAccess 应用必须从以下目录运行：
- `C:\Program Files\`
- `C:\Program Files (x86)\`
- `C:\Windows\`

#### 完整流程总结
1. 修改 manifest 添加 `uiAccess="true"`
2. 对 EXE 签名（商业证书或自签名）
3. 创建安装程序，将软件安装到 `C:\Program Files\SmartClip\`
4. 如果是自签名，需要将证书安装到用户的受信任根证书存储
